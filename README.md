## Возможности для интеграции API с 1С
API дает возможность:
 - Получать информацию и вести учет о складе
 - Управлять удаленно [пользователями](#пользователи) и собирать данные о
   [действиях](#логи) [пользователей](#пользователи)
 - Получать информацию о перечне [полок](#полки) (если склад карусельного типа, то о 
   кассетах) и о доступах каждого пользователя к ним
 - Вести учет того, что хранится на [полках](#места)
 - Удаленно создавать [задачи](#корзины) для пользователей на пополнение и
   выгрузку товаров со склада

***
## Роли
На текущий момент роли жестко зашиты и создаются при первом развертывании приложения. 
Имеется 4 актуальных уровня доступа. При использовании в API, если у метода нет 
требования к роли, то этот метод может использоваться с любым уровнем доступа. 
Рекомендуется использовать только `GET` методы
- id 2 - Оператор. В интерфейсе есть доступ только к вкладкам "Загрузить" и "Выгрузить".
  Может сам формировать себе корзину, но не может получить доступ к полкам из интерфейса, 
  если его ограничить (по умолчанию каждый пользователь при создании получает доступ ко 
  всем полкам)
- id 3 - Администратор. Тот же уровень доступа, что и у оператора, но может создавать, 
  менять и удалять разметку на полке и управлять распределением номенклатурой
- id 4 - Инженер. В интерфейсе есть панель управления полками, пользователями, корзинами 
  у пользователей. Так же может провести инвентаризацию склада
- id 5 - Суперпользователь. Имеет доступ ко всему функционалу

## Пользователи
### Параметры
- **username** - Логин
- **name** - Имя
- **surname** - Фамилия
- **role_id** - id [роли](#роли)
### Начало работы
Скорее всего для вас уже создан пользователь. Чтобы получить токен для взаимодействия 
с требующими доступ методами API используйте `POST` метод `/v1/users/login`, указав 
логин и пароль. Кроме токена этот метод вернет информацию об объекте пользователя. 
`POST` метод `/v1/users/logout` не уничтожает токен, а работает как отметка, что 
взаимодействие этого пользователя с приложением завершено.
### Управление пользователем
При создании нового пользователя `POST` методом `/v1/users/register_new_user` в теле 
запроса не рекомендуется указывать поле `permissions`, т.к. доступы будут выданы в 
зависимости от [role_id](#роли). Обязательно для указания только `username` и 
`password`. Поля `name` и `surname` - имя и фамилия не обязаны быть уникальными и 
могут быть `null`. В заголовке `warehouse_id` указать `1`, если управление лифтовым 
складом, `2`, если карусельным

Для изменения паролей есть два `PUT` метода. `/v1/users/change-password` меняет 
пароль текущего пользователя, а `/v1/users/change-another-password` у другого, если у 
текущего уровень доступа выше. `PUT` метод `/v1/users/{user_id}` позволяет поменять 
уже созданного пользователя.

Для изменения отображения в интерфейсе размещенных на [полке](#полки)
[номенклатур](#номенклатура) можно управлять через `POST` 
`/v1/users/shelf-permission` и `DELETE` `/v1/users/shelf-permission/`, указав в 
`"shelves_id"` список из id полок

### RFID
Если у вас есть доступ к методам `/v1/users/rfid/`, то не рекомендуется их 
использовать удаленно, т.к. они задействуют физическое устройство
***
## Склад
Рекомендуется использовать только `GET` методы для получения данных о складе. Метод 
`/v1/warehouses/{warehouse_id}/shelves` покажет какие [полки](#полки) привязаны к складу.
Метод `/v1/warehouses/get-inventory` даст инженеру данные о том, что хранится на 
физических полках `{"modbus_shelf_id": {"название номенклатуры : единица измерения": 
количество}`. Метод `/v1/warehouses/inventory/download-xlsx` сделает то же самое, но 
вернет готовую xslx таблицу
***
## Номенклатура
### Параметры
 - **name** - Название
 - **description** - Описание
 - **sku** - Артикул
 - **weight** - вес на единицу в граммах
 - **group_id** - id [группы](#группы-номенклатур)
 - **barcode** - штрихкод
 - **measure** - единица измерения. Доступные единицы измерения доступны в `GET` 
   методе `/v1/things/measurement`. Список единиц может дополняться в будущем
 - **length** - устаревшее
 - **width** - устаревшее
 - **height** - устаревшее


Объект **ThingDetailed** содержит в себе параметр **places**, выдавая список [мест](#места)
на которых расположен
### Управление номенклатурой
Создать номенклатуру можно `POST` методом `/v1/things/`, указав необходимые поля. ID 
[группы](#группы-номенклатур) для номенклатуры без группы в устаревших версиях 
рекомендуется указать 1. В обновленной версии будет group_id 1, если поле не указано

Получение общего списка номенклатур можно через `GET` `/v1/things`.

Получение общего списка размещенных и доступных пользователю номенклатур доступно 
через `GET` `/v1/things/read_things_for_user`

Поиск по названию осуществляется методом `GET` `v/1/things/search/`. Параметр 
**placed** отвечает за поиск размещенных номенклатур, а **use_permissions** проверяет и 
размещение и доступ

`GET` `/v1/things/count` выдает общее число номенклатур в базе

Чтобы удалить номенклатуру методом `DELETE` `/v1/things/{thing_id}` нужно убедиться, 
что она нигде не расположена

Так же различными `GET` методами можно посмотреть на каких [местах](#места) и
[полках](#полки) она расположена

### Сканер штрихкодов
Если у вас есть доступ к сканеру штрихкодов, то не рекомендуется использовать 
удаленно методы `/v1/things/barcode/`, т.к. они задействуют физическое устройство

### Принтер этикеток
Если у вас есть доступ к принтеру этикеток, то не рекомендуется использовать 
удаленно методы `/v1/things/print/`, т.к. они задействуют физическое устройство
***
## Группы номенклатур
### Параметры
 - **name** - Название
 - **description** - Описание

### Управление группами номенклатур
По умолчанию существует только одна группа с id `1`. Одна [номенклатур](#номенклатура)
может находиться только в одной группе

Для создания достаточно указать имя и описание в теле `POST` метода `/v1/thing_groups/`

При удалении группы `DELETE` методом у всех привязанных к этой группе с названием `"Без группы"`
[номенклатур](#номенклатура) `group_id` станет `1`
***
## Полки
### Параметры
 - **name** - Название полки
 - **warehouse_id** - `1` если лифт или `2` если карусель
 - **capacity** - вместительность полки в кг
 - **length** - длинна полки
 - **width** - глубина полки
 - **height** - ожидаемая высота (актуально только для каруселей)
 - **dimensions** - габариты полки, записанные с делителем `x`
 - **filled_percentage** - занятая разметкой площадь
 - **modbus_shelf_id** - номер физической полки

### Управление полками
Рекомендуется использовать только `GET` методы в `/v1/shelves/`, т.к. создание, 
обновление и удаление полок рекомендуется только через интерфейс и привязано к параметрам конкретного
физического склада. Здесь можно запросить как список [номенклатур](#номенклатура), так и
[разметку мест](#места) на полке
***
## Места
### Параметры
 - **shelf_id** - id [полки](#полки) 
 - **quantity** - количество [номенклатур](#номенклатура) на этом месте
 - **thing_id** - id [номенклатуры](#номенклатура) на этом месте
 - **warehouse_id** - `1` если лифт или `2` если карусель
 - **area_id** - id [области](#области) при отсутствии после перезапуска панели 
   создаст новую [область](#области) по своим габаритам
 - ~~**length** - длинна (устарело)~~
 - ~~**width** - глубина (устарело)~~
 - ~~**height** - высота (устарело)~~
 - ~~**x** - координата x (устарело)~~
 - ~~**y** - координата y (устарело)~~
 - ~~**z** - координата z (устарело)~~
### Управление местами
Так же рекомендуется использовать только `GET` методы `/v1/places/` для получения 
информации о количестве номенклатур на месте и информации о привязанной
[полке](#полки) и [номенклатуре](#номенклатура)
***
## Области
### Параметры
 - **name** - название области
 - **shelf_id** - id [полки](#полки)
 - **warehouse_id** - `1` если лифт или `2` если карусель
 - **length** - длинна
 - **width** - глубина
 - **height** - высота
 - **x** - координата x
 - **y** - координата y
 - **z** - координата z
 - **places** - информация о привязанных к области [местах](#места)
### Управление областями
Области нужны для отрисовки графики и хранения нескольких номенклатур на одной ячейке 
разметки

И здесь рекомендуется использовать только `GET` методы `/v1/area/`, т.к. правильное 
создание и редактирование безопасно только через интерфейс
***
## Корзины
### Параметры
 - **warehouse_id** - `1` если лифт или `2` если карусель
 - **name** - Название корзины. Обычно `null`
 - **mode** - цель корзины взять `take` или положить `put`
 - **status** - всего 4 статуса `open` - открытая для исполнения, `in_progress` - в 
   работе (в интерфейсе может отображаться как не завершенная, если работа на выдачу не 
   была завершена полностью), `cancelled` - отменена, когда в ней еще была
   [номенклатура](#номенклатура) на выдачу, `done` - завершенная корзина
 - **cart_type** - `internal` (при создании через интерфейс) или `external` 
   (рекомендуется указывать при удаленном создании корзины)
 - **user_created_id** - id [пользователя](#пользователи), которым была создана корзина
 - **user_assigned_id** - id [пользователя](#пользователи), который назначен на 
   исполнение корзины
 - **order_number** - генерируется и кратко описывает созданную корзину
 - **started_at** - дата, когда корзина была перемещена из статуса `open` в `in_progress`
 - **completed_at** - дата, когда корзина была переведена в статус `done`

### Управление корзинами
При создании удаленно `POST` методом `/v1/cart/` для корректной работы используйте 
**status** `open` и **cart_type** `external`

Перед началом добавления предметов в корзину, если создатель корзины будет назначать 
корзину другому [пользователю](#пользователи) лучше сразу использовать `PATCH` метод
`/v1/cart/{cart_id}/assign`

Чтобы добавить [номенклатуру](#номенклатура) в корзину нужно использовать `POST` 
`/v1/cart/{cart_id}/items`. Здесь требуется указать [place_id](#места) и количество 
предметов. При **mode** `take` обратите внимание на количество в указанном
[месте](#места). Так же убедитесь, что у назначенного [пользователя](#пользователи) 
есть доступ к [полке](#полки), где находится указанное [место](#места) и его
[область](#области). После этого указанный пользователь увидит корзину с назначенными 
предметами

Чтобы увидеть корзину с указанной номенклатурой для назначенного пользователя 
используйте `GET` `/v1/cart/map/{cart_id}/{user_id}/by_thing`. Так же здесь вы можете 
увидеть параметр **total_quantity** - это общее количество предметов в корзинах, 
назначенных на это [место](#места) с модом `take`, поэтому будьте аккуратны, если 
**total_quantity** превышает **quantity** у этого [места](#места)
***
## Бэкапы
На данный момент бэкапы может проводить и загружать только [`superuser`](#роли)
***
## Загрузка таблиц
Методом `POST` `/v1/excelfrom-excel-to-db` с уровнем доступа [`engineer`](#роли) можно 
загружать xslx таблицы для создания и дополнения базы [пользователей](#пользователи), 
[номенклатуры](#номенклатура) и [групп номенклатур](#группы-номенклатур)
***
## Модбас
Методы `/v1/modbus/` не предназначены для удаленного использования, т.к. они 
дают взаимодействие с реальной машиной. Так же простое чтение адресов не даст 
полезной информации для учета данных о складе
***
## Конфигурация
В `GET` методе `/v1/config/paid-feauters` есть информация о доступности платных 
возможностей. Остальные методы отвечают за доступ к физическим устройствам
***
## Логи
В методах `GET` `/v1/logs/` вы можете получить информацию об изменяющих состояния 
взаимодействиях со складом по каждому типу объектов или по типам изменений за 
определенный период
